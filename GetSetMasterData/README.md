# GetSetMasterData (セット構成管理 自動同期プロジェクト)

## 概要
ネクストエンジン上のセット商品マスタ情報を自動取得し、構成商品の内訳データを最新の状態に保ちます。

## 開発背景と目的
セット商品の構成内容は頻繁に変更されるものではありませんが、更新を見逃すと出荷ミスや在庫不整合に直結します。そのため、従来は「念のための確認」として毎週手動でCSVをダウンロードしていましたが、本プロジェクトによりその確認作業を完全に自動化しました。

## 主な機能・特徴
- **リスクヘッジとしての自動化**: 人の「作業忘れ」を物理的に排除し、常に最新のセット構成に基づいた運用を担保。
- **フルクラウド連携の完遂**: 認証ライブラリおよび他の取得プロジェクトで培ったノウハウを投入し、安定したデータ連携を実現。
- **マスタ整合性の維持**: 手動更新のタイムラグを無くし、セット商品販売における在庫管理の信頼性を向上。

## 実務上のメリット
「たまにしか起きない変更」のために毎週のリソースを割く無駄を省き、システムがバックグラウンドで監視し続ける体制を構築しました。これにより、現場スタッフは「マスタが古いかもしれない」という不安から解放され、本来の業務に集中できる環境を実現しています。

## 技術的な特徴
- **堅牢なエラーハンドリング**: 指数バックオフを用いた自動リトライ機能を備え、一時的なネットワークエラーによる停止を防ぎます。
- **柔軟なログ制御**: 運用状況に合わせてログの出力密度を3段階で調整可能です。
- **保守性の高いコード構造**: 全関数にJSDocを完備し、Google Apps Scriptエディタ上での開発・デバッグをサポートします。

## セットアップ方法

### 1. スクリプトプロパティの設定
Google Apps Script の「プロジェクトの設定」にて、以下のプロパティを設定してください。

| キー | 内容 | 備考 |
| :--- | :--- | :--- |
| SPREADSHEET_ID | 書き込み先のスプレッドシートID | |
| SHEET_NAME | 書き込み先のシート名 | |
| LOG_LEVEL | 1(詳細), 2(通常), 3(なし) | 推奨: 2 |
| CLIENT_ID | NextEngine API クライアントID | |
| CLIENT_SECRET | NextEngine API クライアントシークレット | |
| REDIRECT_URI | 認証用リダイレクトURL | |
| NOTIFICATION_EMAIL | 通知先メールアドレス | 未設定時は実行者 |

### 2. 認証の実行
1. `01_設定管理.gs` の `showAllConfig()` を実行して設定を確認します。
2. 認証URL生成関数を実行し、ブラウザでNextEngineの認可を行ってください。

### 3. 自動実行の設定
`08_トリガー管理.gs` の `setupRecommendedTriggers()` を実行すると、以下のスケジュールで自動実行が設定されます。
- **午前2時**: 認証トークンの自動更新
- **午前3時**: セット商品マスタの同期実行

## ファイル構成

| ファイル名 | 役割 |
| :--- | :--- |
| `01_設定管理.gs` | スクリプトプロパティの取得・管理 |
| `02_スプレッドシート操作.gs` | スプレッドシートへの書き込み、ヘッダー管理 |
| `03_ログ出力制御.gs` | ログレベルに応じたコンソール出力の制御 |
| `04_API呼び出し.gs` | NextEngine APIとの通信、ページネーション処理 |
| `05_メイン処理.gs` | 全体の実行フロー制御（メイン関数: `updateSetGoodsMaster`） |
| `06_エラー通知.gs` | エラー発生時のメール通知、サマリー送信 |
| `07_リトライ処理.gs` | 指数バックオフを用いた再試行ロジック |
| `08_トリガー管理.gs` | 定期実行トリガーの作成・削除・一覧管理 |
