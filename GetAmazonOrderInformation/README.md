# NE_(モール名)受注情報取得

ネクストエンジンのAPIを使用して、(モール名)受注データ（個人情報を含む）を  
Googleスプレッドシートへ自動転記するGoogle Apps Script（GAS）プロジェクトです。

---

## 背景・目的

従来は必要な情報をCSVでCSからダウンロードし、手動でスプレッドシートに転記していたが、
作業自動化により手動転記作業をなくし、工数削減と転記ミスの防止を実現する。

---

## スクリプトプロパティ一覧

GASエディタの「プロジェクトの設定」→「スクリプト プロパティ」に以下を設定してください。

| キー | 説明 | 必須 |
|---|---|:---:|
| `CLIENT_ID` | ネクストエンジンアプリのクライアントID | ✅ |
| `CLIENT_SECRET` | ネクストエンジンアプリのクライアントシークレット | ✅ |
| `REDIRECT_URI` | WebアプリのデプロイURL | ✅ |
| `ACCESS_TOKEN` | APIアクセストークン（認証後に自動保存） | 自動 |
| `REFRESH_TOKEN` | リフレッシュトークン（認証後に自動保存） | 自動 |
| `TOKEN_OBTAINED_AT` | トークン取得日時・ミリ秒（自動保存） | 自動 |
| `TOKEN_UPDATED_AT` | トークン最終更新日時・ミリ秒（自動保存） | 自動 |
| `OUTPUT_SPREADSHEET_ID` | 出力先スプレッドシートID | ✅ |
| `OUTPUT_SHEET_NAME` | 出力先シートタブ名（未設定時は `API`） | － |
| `LOG_SHEET_NAME` | ログシートタブ名（未設定時は `LOG`） | － |
| `NOTIFY_EMAIL` | エラー通知先メールアドレス | ✅ |
| `DEVELOPMENT_MODE` | `"true"` で開発モード（未設定または `"false"` で本番） | － |

## 出力先スプレッドシートの構成

### API シート（受注データ）

毎日の実行で出荷確定済み受注データが追記されます。  

- **自動ヘッダー生成**: シートが空の場合、実行時に自動でヘッダー（オレンジ背景 [#e67e22] / 白文字）が生成されます。
- **0落ち防止策**: 電話番号や郵便番号などのデータはすべて文字列（String）として書き込まれます。スプレッドシートによる意図しない数値変換を防ぎます。

| 列 | ヘッダー名 | APIフィールド名 |
|---|---|---|
| A | 伝票番号 | `receive_order_id` |
| B | 受注日 | `receive_order_date` |
| C | 出荷予定日 | `receive_order_send_plan_date` |
| D | 購入者名 | `receive_order_purchaser_name` |
| E | 購入者郵便番号 | `receive_order_purchaser_zip_code` |
| F | 購入者住所1 | `receive_order_purchaser_address1` |
| G | 購入者住所2 | `receive_order_purchaser_address2` |
| H | 購入者電話番号 | `receive_order_purchaser_tel` |
| I | 購入者メールアドレス | `receive_order_purchaser_mail_address` |
| J | 送り先名 | `receive_order_consignee_name` |
| K | 送り先郵便番号 | `receive_order_consignee_zip_code` |
| L | 送り先住所1 | `receive_order_consignee_address1` |
| M | 送り先住所2 | `receive_order_consignee_address2` |
| N | 送り先電話番号 | `receive_order_consignee_tel` |

### LOG シート（エラー・異常ログ）

**正常実行時はログを記録しません。**  
以下の場合のみ記録されます。

| 実行結果 | 記録条件 | メール通知 |
|---|---|:---:|
| `0件` | 取得件数が0件だった場合 | なし |
| `エラー` | 処理中に例外が発生した場合 | あり |

| 列 | ヘッダー名 | 内容 |
|---|---|---|
| A | 実行日時 | `yyyy-MM-dd HH:mm:ss` 形式 |
| B | 対象日付 | 取得対象の出荷確定日 |
| C | 実行関数名 | `dailyRun` / `manualRun` |
| D | 取得件数 | 数値（0件の場合も記録） |
| E | 実行結果 | `0件` / `エラー` |
| F | エラー内容 | エラー時のみ記載 |

## 初回セットアップ手順

### 1. ライブラリの追加

GASエディタの左メニュー「ライブラリ」→「＋」をクリックし、以下の設定で追加します。

- スクリプトID: NEAuthプロジェクトのスクリプトID
- バージョン: 最新を選択
- 識別子: `NEAuth`

### 2. Webアプリとしてデプロイ

「デプロイ」→「新しいデプロイ」→「Webアプリ」を選択し、デプロイURLを取得します。  
取得したURLをスクリプトプロパティ `REDIRECT_URI` に設定します。

### 3. スクリプトプロパティの設定

上記「スクリプトプロパティ一覧」を参照し、必須項目をすべて設定します。

### 4. 初回認証

GASエディタで `testGenerateAuthUrl()` を実行し、表示されたURLをブラウザで開きます。  
ネクストエンジンにログインすると `doGet()` が自動実行され、トークンが保存されます。

### 5. API接続確認

`testApiConnection()` を実行し、ユーザー情報が取得できれば認証完了です。

### 6. LOGシートの初期化

`testPhase5_1()` を実行し、スプレッドシートにLOGタブが作成されることを確認します。

### 7. トリガーの設定

GASエディタの「トリガー」から以下の2つを設定します。

| 関数名 | タイプ | 推奨時刻 |
|---|---|---|
| `dailyTokenRefresh` | 日付ベースのタイマー | 午前3時〜4時 |
| `dailyRun` | 日付ベースのタイマー | 午前3時〜4時 |

> **重要:** `dailyTokenRefresh` はトークンの有効期限（3日）を延長し続けるため、  
> `dailyRun` の実行頻度に関わらず**毎日実行が必須**です。

## 主要関数の説明

### 日常運用

| 関数名 | 説明 |
|---|---|
| `dailyRun()` | 前日の出荷確定済み受注を取得してシートに追記（トリガー用） |
| `manualRun('YYYY-MM-DD')` | 任意の出荷確定日を指定して手動実行（引数省略時は前日） |
| `dailyTokenRefresh()` | トークンの有効期限を延長（毎日トリガー実行） |

### 確認・トラブルシューティング

| 関数名 | 説明 |
|---|---|
| `testApiConnection()` | トークンの有効性とAPI接続を確認 |
| `showMyTokens()` | 保存されているトークン情報と取得日時を表示 |
| `testGenerateAuthUrl()` | 再認証用URLを生成（トークン切れ時に使用） |
| `clearMyTokens()` | 保存済みトークンをすべて削除（再認証の前準備） |

### 開発・テスト

| 関数名 | 説明 |
|---|---|
| `testRun()` | 開発モード（楽天市場店）でデータ取得のみ確認（シート書き込みなし） |
| `testPhase5_1()` | LOGシートの初期化とダミーログ書き込みを確認 |
| `testPhase5_2()` | リトライ処理の動作をコンソールログで確認 |
| `testPhase5_3()` | 開発モードで `manualRun()` を実行し統合動作を確認 |

## 環境切替（本番 / 開発）

スクリプトプロパティ `DEVELOPMENT_MODE` で対象店舗を切り替えます。

| 値 | 対象店舗 | 用途 |
|---|---|---|
| 未設定 または `"false"` | モールA（店舗ID: 20） | 本番運用 |
| `"true"` | モールB（店舗ID: 2） | 開発・テスト |

> テスト用関数（`testRun` / `testPhase5_2` / `testPhase5_3`）は  
> 実行中のみ自動的に開発モードへ切り替わり、終了後に元の状態へ戻します。

## API仕様・重要事項

### エンドポイント

| 項目 | 値 |
|---|---|
| ホスト | `https://api.next-engine.org` |
| エンドポイント | `/api_v1_receiveorder_base/search` |
| メソッド | POST |
| 1回の最大取得件数 | 1,000件 |

### トークン管理

- ネクストエンジンはAPIコールのたびにトークンをローテーションします
- **APIコール時は `access_token` と `refresh_token` の両方をペイロードに含めること**
- **レスポンスの新しいトークンを毎回スクリプトプロパティへ保存すること**
- 上記を怠ると次回のAPIコールで認証エラーになります

### ページネーション

- レスポンスの `count` フィールドはページ内件数であり総件数ではありません
- 終了判定は「取得件数 < 1,000件」で行います

### リトライ処理

一時的なネットワークエラーやサーバー負荷エラーに対して、APIクォータを考慮した自動リトライを行います。

| 設定 | 値 |
|---|---|
| 最大リトライ回数 | 3回 |
| ウェイト方式 | 指数バックオフ（2のn乗倍: 1秒 → 2秒 → 4秒） |
| リトライ対象外 | 認証エラー（tokenを含むエラーメッセージ時は即時中断） |
| リトライ対象外 | 認証エラー（即時スロー） |

## トラブルシューティング

### エラー通知メールが届いた場合

通知メールには発生時刻・関数名・エラー詳細が含まれています。内容に応じて以下のリカバリを行ってください。

1. **認証エラー (`token error` 等)**:
   - 原因: トークンの有効期限切れ、または他のアプリによるトークンの上書き。
   - 対処: `testGenerateAuthUrl()` を実行して再認証を行ってください。
2. **APIリクエストエラー / リトライ上限到達**:
   - 原因: ネクストエンジン側の一時的な高負荷、または通信の不安定。
   - 対処: 少し時間を置いてから `manualRun('YYYY-MM-DD')` で対象日のデータを再取得してください。
3. **スプレッドシート参照エラー**:
   - 原因: シート名やIDの変更、または閲覧権限の喪失。
   - 対処: スクリプトプロパティ `OUTPUT_SPREADSHEET_ID` 等の再設定を確認してください。

### LOGシートに `0件` が記録されている場合

対象日に出荷確定済み受注が存在しなかった可能性があります（祝日・休業日など）。  
ネクストエンジン管理画面で対象日の出荷件数を確認してください。

### データが取得できない場合

- `testApiConnection()` でトークンの有効性を確認
- `showMyTokens()` でトークンの取得日時を確認
- スクリプトプロパティ `OUTPUT_SPREADSHEET_ID` と `OUTPUT_SHEET_NAME` が正しく設定されているか確認
